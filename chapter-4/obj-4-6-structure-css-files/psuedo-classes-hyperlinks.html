Testing order of CSS rules on pseudoclasses

Case 1:

a.case1:link {
color: blue;
}
a.case1:hover {
color: white;
background-color: black;
}
a.case1:visited {
color: purple;
}
 Link here displays case 1

After clicking the link, when you hover over it it will have purple text on a black background

Case 2:

a.case2:link {
color: blue;
}
a.case2:visited {
color: purple;
}
a.case2:hover {
color: white;
background-color: black;
}
 Link here displays case 2

After clicking the link, when you hover over it it will have white text on a black background

This styling is unaffected by the link being visited or not

The different behaviours depend on the order of CSS rules:

  • In Case 1 the rule for the :visited pseudo-class comes last in the embedded styles, so overwrites the hover rule for the color property.
  • In Case 2 the rule of hover comes last, so all anchor tags behave the same when they're hovered over; styles using :hover aren't overwritten by styles later in the CSS
